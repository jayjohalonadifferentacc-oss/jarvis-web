JARVIS

A personal AI assistant whose memory is a folder of markdown files — and whose brain is a live visualisation of that memory actually working. Runs locally and free on your own machine. No API key in the primary path. No bill.

What it is

Three surfaces over one foundation

Chat, a living brain, and live ops — all reading the same markdown memory and the same event stream.

Chat — text or voice

Talk or type. Local speech synthesis (Kokoro) and push-to-talk, both routed through the same gate as everything else.

The Brain

A Three.js tangle that animates genuine memory activity. Never a fake particle — if nothing is happening, nothing moves.

Ops

Live stats derived from the same markdown and event log — plus a single choke point that asks before anything irreversible happens.

Local-first

Your memory is a plain folder of markdown you can read and edit in any text editor. Swaps models without touching the brain.

The Gate

Every tool call is risk-rated. Destructive actions always ask first, in plain English, before they happen. It fails closed.

On your phone

The Android app drives your PC's JARVIS over the LAN, or add it to your home screen. Same brain, wherever you are.

Windows

JARVIS_3.0.0_x64-setup.exe

Native Windows app (Tauri 2). Installs a frameless dark window with system tray, global Alt+J hotkey, and runs the assistant fully on your machine.
Download installer ~82 MB · x64
▶ Install steps
  1. Download JARVIS_3.0.0_x64-setup.exe above.
  2. Run it. Windows SmartScreen may say "protected your PC" — click More info → Run anyway (it's a self-signed sideload, not malware).
  3. Finish the installer. JARVIS launches to the system tray.
  4. First run needs a local model: install Ollama, then ollama pull qwen3:4b.
RECOMMENDED

Android

JARVIS-3.0.0-arm64-release.apk

Thin webview client for your phone. It shows a "connect to PC" screen, then drives your PC's JARVIS over the LAN — there's no brain on the device itself. Requires JARVIS running with remote mode on.
Download APK ~4.9 MB · arm64
▶ Install steps
  1. Download JARVIS-3.0.0-arm64-release.apk above and open it on your phone.
  2. Allow "install from unknown source" when Android prompts (it's self-signed).
  3. On your PC, set %APPDATA%\com.jay.jarvis\desktop.json to {"remote":true} and restart JARVIS.
  4. In the PC HUD, click PAIR, allow port 3210 through Windows Firewall, then scan the QR / enter the URL in the app.

Connect your phone to your PC

JARVIS on Android is a window into the copy running on your Windows machine. Turn on remote mode once, pair with a QR, and your phone drives the same brain from anywhere on your Wi-Fi.

desktop.json → {"remote": true}

Phone won't connect? The usual three

  • Installed over an old APK? The app is now signed with a fresh key, so Android refuses to update in place. Uninstall the old JARVIS app first, then install the new APK. Your saved address is wiped by that anyway — scan the PAIR QR again on first launch.
  • "Can't reach that address" / spin forever? The phone and PC must be on the same Wi-Fi, and port 3210 must be open. Run the enable-lan.bat from the download (right-click → Run as administrator) once, or open an admin PowerShell and allow it: New-NetFirewallRule -DisplayName "JARVIS LAN" -Direction Inbound -LocalPort 3210 -Protocol TCP -Action Allow -Profile Private
  • "Connected, but not paired"? The QR you scanned is stale. Re-open JARVIS on the PC, click PAIR again, and scan the fresh QR (or paste its full ?pair=… URL into the app). A re-scan always overrides a saved address.

JARVIS on Android is a remote window into your PC — there is no brain on the phone. If the PC is off or asleep, the app can do nothing, by design.
